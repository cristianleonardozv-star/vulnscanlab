Principales herramientas de Kali Linux

Selecciona una herramienta para explorar su documentación, comandos y ejemplos de uso en redes locales.

Reconocimiento

Nmap

Escáner de red y puertos. Identifica hosts activos, servicios en ejecución y sistemas operativos dentro de una red local.

nmap -sV 192.168.1.0/24
Reconocimiento

Netdiscover

Descubre dispositivos activos en la red local utilizando peticiones ARP activas y pasivas.

netdiscover -r 192.168.1.0/24
Tráfico de red

Wireshark

Captura y analiza paquetes de red en tiempo real. Permite inspeccionar protocolos, conexiones y datos transmitidos.

wireshark -i eth0
Tráfico de red

Tcpdump

Captura de tráfico de red desde la línea de comandos. Ideal para filtrar y guardar paquetes de forma automatizada.

tcpdump -i eth0 port 80
Seguridad web

Nikto

Escáner de servidores web que detecta configuraciones inseguras, software desactualizado y rutas peligrosas.

nikto -h 192.168.1.10
Seguridad web

Burp Suite

Proxy de interceptación para analizar y manipular peticiones HTTP/HTTPS. Herramienta clave en auditorías web.

burpsuite &
Vulnerabilidades

OpenVAS / GVM

Sistema integral de gestión y evaluación de vulnerabilidades con base de datos de CVEs actualizada automáticamente.

gvm-start
Explotación

Metasploit

Framework de explotación de vulnerabilidades conocidas. Organizado en módulos: exploits, payloads y auxiliares.

msfconsole
Explotación

Sqlmap

Detección y explotación automática de inyecciones SQL en aplicaciones web. Soporta múltiples bases de datos.

sqlmap -u "http://target/id=1"
Explotación

Hydra

Herramienta de fuerza bruta para servicios de red: SSH, FTP, HTTP, SMB. Evalúa la robustez de contraseñas.

hydra -l admin -P wordlist.txt ssh://192.168.1.1
Wi-Fi

Aircrack-ng

Suite para auditoría de redes inalámbricas: captura de handshakes WPA, análisis de paquetes y evaluación de seguridad Wi-Fi.

airodump-ng wlan0mon
Redes

Netcat

La "navaja suiza" de las redes. Transferencia de datos, diagnóstico de puertos y conexiones TCP/UDP desde terminal.

nc -lvnp 4444
Monitoreo

Grafana

Plataforma de visualización y análisis de métricas. Permite crear dashboards en tiempo real para monitorear el comportamiento de la red y detectar anomalías de seguridad.

grafana-server start
Monitoreo

Zabbix

Sistema de monitoreo de red empresarial de código abierto. Supervisa disponibilidad, rendimiento y seguridad de hosts, servicios y dispositivos de red en tiempo real.

zabbix_server -c /etc/zabbix/zabbix_server.conf
Explotación

Deephat

Herramienta de análisis de vulnerabilidades orientada a la identificación de vectores de ataque mediante técnicas de enumeración profunda en redes locales.

deephat --scan --target 192.168.1.0/24

¿Cómo se realiza una auditoría de red?

El proceso de detección de vulnerabilidades sigue fases definidas. Cada herramienta cumple un rol específico.

01

Reconocimiento

Identificar qué dispositivos, puertos y servicios están activos en la red. Herramientas: Nmap, Netdiscover.

02

Captura de tráfico

Analizar los datos que circulan por la red para detectar comunicaciones inseguras. Herramientas: Wireshark, Tcpdump.

03

Análisis de vulnerabilidades

Identificar debilidades en servicios y aplicaciones web. Herramientas: Nikto, OpenVAS.

04

Reporte de hallazgos

Documentar cada vulnerabilidad encontrada con su nivel de riesgo y recomendaciones de mitigación.

Términos clave

Conceptos esenciales para entender la seguridad en redes locales.

CVE
Common Vulnerabilities and Exposures. Identificador único para vulnerabilidades de seguridad públicamente conocidas.
Puerto (Port)
Número lógico (0-65535) que identifica un servicio de red en un dispositivo. Ej: puerto 80 = HTTP.
Penetration Testing
Prueba de penetración autorizada: simulación de ataques para identificar vulnerabilidades antes que los atacantes.
Payload
Código que se ejecuta en el sistema objetivo durante una explotación. Puede ser una shell reversa, un agente, etc.
Sniffing
Captura y análisis del tráfico de red para inspeccionar los datos que viajan entre dispositivos.
ARP
Address Resolution Protocol. Protocolo que asocia direcciones IP con direcciones MAC en una red local.

Uso responsable del conocimiento

La ciberseguridad ética tiene reglas claras que todo profesional debe conocer.

¿Qué es el Hacking Ético?

El hacking ético consiste en utilizar las mismas técnicas y herramientas que un atacante malicioso, pero con autorización expresa del propietario del sistema y con el objetivo de mejorar su seguridad. En Colombia, la Ley 1273 de 2009 tipifica como delito el acceso no autorizado a sistemas informáticos. Usar estas herramientas sin permiso puede tener consecuencias legales graves.
